Node.js

Googles V8 Javascript-engine med et ekstra API som tillater oss å bruke Javascript som et serversidespråk.

Ettersom det er Javascript kan man bruke erfaringer fra nettleseren.

I enkelte tilfeller kan man også dele kode mellom tjener og klient.

Som med Javascript for vi et ikke-blokkerende (asynkront) miljø, uten trådhåndtering.

Kjappheten til V8 og den asynkrone modellen gjør at Node.js egner seg godt til input og output operasjoner.

Det som gjør det kjapt for I/O, gjør det ikke så godt egnet til tunge CPU operasjoner

Disse vil da blokkere kjøringen, og det gjør at vi mister fordelen med asynkronitet.


// get standard libraries
var fs = require('fs');
var http = require('http');

// create HTTP server on port 3000
http.createServer(function (req, res){
  fs.createReadStream('file.dat').pipe(res);
}).listen(3000);

Massivt økosystem av kode

Har et utrolig voksende community hvor fokuset er stort på modulæritet og kode-deling.

Har nå over 90 000 pakker delt.

Med over 7 000 000 nedlastninger om dagen.

«Felles filosofi» for Node.js brukere er å lage små delte pakker som kan gjenbrukes.

Søk på f.eks google etter node mac notification center gir node-notifier.


$ npm install --save node-notifier


// File: test.js
var Notification = require('node-notifier');

new Notification().notify({
  message: 'Hello World!'
}); // Sends a message to OS X


$ node test.js
