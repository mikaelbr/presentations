Node.js

Googles V8 Javascript-engine med et ekstra API som tillater oss å bruke Javascript som et serversidespråk.

Først lansert i 2009 av Ryan Dahl, som på den tiden, såvidt jeg husker, jobbet for Joyent.

Ettersom det er Javascript kan man bruke erfaringer fra nettleseren.

I enkelte tilfeller kan man også dele kode mellom tjener og klient.

Eksempler på det man kan dele er domenelogikk, validering, modeller eller ligende.

Som med Javascript for vi et ikke-blokkerende (asynkront) miljø, uten trådhåndtering.

Kjappheten til V8 og den asynkrone modellen gjør at Node.js egner seg godt til input og output operasjoner.

I Javascript har man en eventloop med flere steg som håndterer forskjellig typer kode.

Du kan utsette enkelte operasjoner og ta de igjen senere i en eventloop.

I mellomtiden da kan man gjøre andre operasjoner.

Eksempelvis, kan man starte å lese fra en fil, men ikke håndtere dette før innholdet er klart.

Det som gjør det kjapt for I/O, gjør det ikke så godt egnet til tunge CPU operasjoner

Disse vil da blokkere kjøringen, og det gjør at vi mister fordelen med asynkronitet.

Dersom du opptar CPU vil du ikke ha noe effekt med asynkronitet.


// get standard libraries
var fs = require('fs');
var http = require('http');

// create HTTP server on port 3000
http.createServer(function (req, res){
  fs.createReadStream('file.dat').pipe(res);
}).listen(3000);

Her ser vi et eksempel på hvordan vi kan opprette en HTTP tjener som streamer en fil til klienten.

Vi ser at vi kan gjøre ting på et litt lavere nivå om vi ønsker det. Det finnes også API-endepunkt for rene TCP-kall.

Massivt økosystem av kode

Har et utrolig voksende community hvor modulæritet og kode-deling er i fokus.

Det finnes pakker for å gjøre det meste, og dette er ikke begrenset til tooling, eller webutvikling, men det er også kode for roboter, kamera-integreringer, moduler for Nevrale Nett og maskinlæring.

Men det og også veldig mange pakker for asset håndtering som kompilering av LESS, SASS, CoffeeScript eller lignende.

Har nå over 90 000 pakker delt.

Med over 7 000 000 nedlastninger om dagen.

«Felles filosofi» for Node.js brukere er å lage små delte pakker som kan gjenbrukes.

Fordelen er at alle pakker i utgangspunktet er lokale til prosjektet. Det betyr at to prosjekter kan være avhengig av to forskjellige versjoner av et bibliotek.

Dette er i motsetning til f.eks PIP/easy_install i Python.

Søk på f.eks google etter node mac notification center gir node-notifier.

Et kjapt blikk på dokumentasjonen ser vi hvordan vi kan bruke det.


$ npm install --save node-notifier


// File: test.js
var Notification = require('node-notifier');

new Notification().notify({
  message: 'Hello, World!'
}); // Sends a message to OS X


$ node test.js

Dette vil sende en melding til Notification Center på Mac.

Hadde også fungert på Linux og på Windows med Growl.
